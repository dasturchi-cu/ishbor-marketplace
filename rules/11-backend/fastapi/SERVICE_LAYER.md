# SERVICE_LAYER.md

**Project:** Ishbor Marketplace  
**Pattern:** Application services orchestrating repositories, cache, outbox, and external adapters  
**Module map source:** [BACKEND_ARCHITECTURE.md](../BACKEND_ARCHITECTURE.md) §3

---

## 1. Purpose

Each service class is a bounded context entry point. Routers and Celery tasks call services — never repositories directly (except UnitOfWork internals). Services enforce business rules, manage PostgreSQL transactions, emit domain events to the outbox, and trigger cache invalidation.

**Conventions:**
- Methods are `async` unless running in Celery sync context (use `run_async()` bridge)
- Money methods require `idempotency_key: str`
- Return Pydantic DTOs from `app/schemas/`, not ORM models
- Raise `IshborHTTPException` with Uzbek messages for user-facing errors
